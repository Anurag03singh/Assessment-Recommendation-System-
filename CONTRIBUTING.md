# Contributing Guidelines

Thank you for your interest in contributing to the SHL Assessment Recommendation System!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install dependencies: `pip install -r requirements.txt`
4. Build the index: `cd backend && python embeddings.py`
5. Run tests: `python evaluate.py`

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and modular

## Making Changes

1. Create a new branch: `git checkout -b feature/your-feature`
2. Make your changes
3. Test thoroughly
4. Commit with clear messages: `git commit -m "Add feature X"`
5. Push to your fork: `git push origin feature/your-feature`
6. Create a Pull Request

## Testing

Before submitting:
- Run the evaluation script
- Test the API endpoints
- Verify the frontend works
- Check for any errors in logs

## Areas for Contribution

- Improving recommendation accuracy
- Adding new assessment sources
- Enhancing the frontend UI
- Optimizing performance
- Adding more evaluation metrics
- Improving documentation

## Questions?

Feel free to open an issue for any questions or suggestions.
